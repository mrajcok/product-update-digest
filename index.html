REST Collector Diagnostics — Now easier than ever! | Cribl

Cribl has released new debugging tools for its REST Collector, improving visibility into potential issues when collecting data from HTTP APIs.

  • The new REST Collector Diagnostics allow users to run collector jobs in debug mode and interactively inspect request and response details – including headers, payloads, URLs, and status codes.
  • Diagnostics provide visibility into authentication, discovery, and collection phases, pinpointing failures like authentication errors or unexpected data formats.
  • A generated curl command is included, enabling users to replicate requests for testing in external tools like Postman.
  • Diagnostic data is encrypted, stored locally, and features built-in redaction to protect sensitive information; it is not ingested into Cribl.

The hidden infrastructure challenge of agentic AI

Ocient highlights the increased infrastructure demands created by agentic AI systems due to their iterative and parallel query nature.

  • OcientAIQ™ is positioned as a solution designed to handle the increased workload from autonomous agents, which decompose tasks into numerous sub-questions and continuously iterate.
  • The article argues that traditional approaches like semantic layers and governance frameworks don't address the fundamental issue of query volume generated by agents.

Ocient blog · 2026-06-05

Graph analytics at scale without Spark bottlenecks

OcientAIQ™ now features OCGraph, a library for running large-scale graph algorithms directly within the platform, eliminating Spark bottlenecks.

  • OCGraph is available in Java (as part of the Ocient JDBC driver – com.ocient:ocient-jdbc4) and Python (ocient-graph via PyPI). Both libraries offer identical sets of algorithms.
  • Graphs are represented as two standard Ocient tables: a vertices table (with 'id' as a required column) and an edges table (with 'srcid' and 'destid' as required columns). This allows leveraging existing Ocient table features like indexing and joins.
  • The library includes algorithms such as Connected Components, PageRank, Triangle Count, Weighted Shortest Paths, and Label Propagation, offering comparable functionality to Spark’s GraphX/GraphFrames with a simplified API.
  • Benchmarks on the LDBC Graphalytics XL twitter_mpi dataset (52.6M vertices, 1.96B edges) show OCGraph achieving performance improvements of 7x to 17x for iterative algorithms like PageRank, Connected Components, and Label Propagation, with successful (and faster) completion of triangle count where Spark failed.

Cert’s up: Using self-signed TLS to encrypt Cribl-to-Cribl data streams | Cribl

Cribl Stream and Edge Nodes now support encrypting data streams using self-signed TLS certificates, offering an alternative to cloud-managed certificates.

  • The process involves creating a Certificate Authority (CA), generating a certificate for the Worker Node, defining Subject Alternative Names (SANs) for hostname and IP address resolution, and signing the certificate with the CA.
  • Specific OpenSSL commands are provided for generating keys, CSRs, and signing certificates, including recommended settings for certificate validity periods (e.g., 825 days to avoid Apple device limitations).
  • Configuration requires placing the CA certificate on the Edge Node and the worker certificate within the Stream Worker Group settings, followed by committing and deploying the changes.
  • The article emphasizes the importance of SANs for modern TLS validation and provides guidance on file permissions (chmod 600) to maintain security.

Managing data lake data: best practices for scalability and efficiency | Cribl

Managing Data Lake Data: Best Practices for Scalability and Efficiency

This article details best practices for managing data lakes to avoid them becoming costly and unusable “data swamps.” The core claim is that controlling data before it enters the lake—through filtering, enrichment, masking, and routing—is the most effective strategy for scalability.

  • Zonal Architecture: Organizing data into layers based on refinement level (raw, processed, etc.) preserves data lineage and enforces quality stages.
  • Metadata-First Governance: Creating metadata at ingestion (data catalog, lineage tracking, access control) ensures data discoverability and trust.
  • Storage & Query Optimization: Utilizing columnar formats (Parquet, ORC), partitioning, and compaction improves query performance and reduces compute costs.
  • Automated Lifecycle Management: Tiered storage, retention policies, and automated archival/deletion control costs by aligning storage with data value over time.

Cribl addresses this challenge with Cribl Stream (upstream control for filtering/reducing/enriching data before storage) and Cribl Lake (schema-on-read storage for investigation and analytics).

Cribl highlights a critical issue with AI deployments: lack of real-time, complete data access, leading to confident but incorrect decisions.

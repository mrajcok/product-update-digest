Cert’s up: Using self-signed TLS to encrypt Cribl-to-Cribl data streams | Cribl

Cribl Stream and Edge Nodes can now utilize self-signed TLS certificates for encrypted data transmission.

  • The process involves creating a Certificate Authority (CA), generating a certificate for the Worker Node, and defining Subject Alternative Names (SANs) for the certificate.
  • Specifically, openssl commands are used to generate keys, Certificate Signing Requests (CSRs), and sign certificates, with example commands provided for creating a CA (openssl genpkey), a worker CSR (openssl req), and signing the CSR (openssl x509).
  • Configuration requires placing the CA certificate (ca.crt) on the Edge Node and the worker key/certificate (e.g., worker.key, worker.crt) on the Stream Worker, with explicit instructions for file locations and permissions (chmod 600).
  • The article highlights the importance of including SANs in the certificate configuration to ensure compatibility with modern TLS validation, and recommends a certificate validity period of 825 days to avoid Apple device compatibility issues.

Managing data lake data: best practices for scalability and efficiency | Cribl

Managing Data Lake Data: Best Practices for Scalability and Efficiency

This article details best practices for managing data lakes to avoid them becoming costly and unusable “data swamps.” The core claim is that controlling data before it enters the lake—through filtering, enrichment, masking, and routing—is the most effective strategy for scalability.

  • Zonal Architecture: Organizing data into layers based on refinement level (raw, processed, etc.) preserves data lineage and enforces quality stages.
  • Metadata-First Governance: Creating metadata at ingestion (data catalog, lineage tracking, access control) ensures data discoverability and trust.
  • Storage & Query Optimization: Utilizing columnar formats (Parquet, ORC), partitioning, and compaction improves query performance and reduces compute costs.
  • Automated Lifecycle Management: Tiered storage, retention policies, and automated archival/deletion control costs by aligning storage with data value over time.

Cribl addresses this challenge with Cribl Stream (upstream control for filtering/reducing/enriching data before storage) and Cribl Lake (schema-on-read storage for investigation and analytics).

When your telemetry meets AI. (It usually fails.) | Cribl

Cribl highlights the necessity of adapting telemetry infrastructure for modern AI-driven operations, claiming that traditional systems optimized for human analysis struggle with the demands of continuous machine analysis.

  • Cribl advocates for “structured at ingest,” parsing, normalizing, and enriching data before storage to avoid errors caused by systems inferring meaning from ambiguous data.
  • The blog details how decoupling ingest, processing, routing, and storage enables cost control without sacrificing data coverage, addressing the economic limitations of legacy platforms facing high-frequency queries.
  • Cribl emphasizes embedding operational knowledge (runbooks, policies, etc.) directly into telemetry streams, fusing machine signals with human rules for more contextualized and valuable analysis.
  • A key architectural component is governance and policy enforcement integrated within the data pipeline, not as an afterthought, and designing for cost-awareness throughout the entire process.

Federated security observability: Investigating bot traffic with Cribl Search and Hydrolix | Cribl

Cribl Search can now federate queries across Cribl Lake and Hydrolix to investigate issues like bot traffic causing cache busting.

  • Analysts can correlate origin application logs (Cribl Lake) with CDN logs (Hydrolix) using Cribl Search’s union operator to identify traffic patterns indicative of bots.
  • The example scenario focuses on detecting bots that overload an inventory API by bypassing the cache with randomized query parameters (e.g., cb=928374).
  • Hydrolix provides cost-effective, long-term storage of full-fidelity CDN telemetry, including client IP, user agent, cache status, and response timing.
  • Cribl CoPilot can automatically summarize investigations into narratives, and queries can be documented in Cribl Search Notebooks for sharing.
